# Resonance SDK

Welcome to the **Resonance SDK**. This library serves as the official bridge between **Team HI** (Python Frontend) and **Our Team** (C++ Core Plate Drivers).

## What Does This Do?
Team HI works in Python, while the physical Chladni hardware drivers are written in C++ and require strict JSON commands. 

This SDK hides the complexity of networking and JSON formatting. It provides a simple, pre-compiled C++ library (`libresonance_sdk.so`) that Team HI can call directly from Python using `ctypes`.

### The Workflow
1. **Team HI (Python):** Sends simple variables (like `volume`, `bpm`, `symbol_id`) to the SDK.
2. **SDK (C++):** Wraps those variables into perfectly formatted JSON and generates reliable UNIX timestamps.
3. **ZeroMQ (Network):** Instantly pushes the payload over the local network via IPC to the physical hardware receiver (`rx_driver`).

## How to Compile Everything (For Core & Setup)
You must compile this SDK into a shared library (`.so`) before Python can use it. 
Run these commands in a WSL (Linux) terminal:

```bash
# 1. Install required dependencies
sudo apt update
sudo apt install -y build-essential cmake libzmq3-dev

# 2. Build the shared library (.so)
cd code/resonance_sdk
mkdir -p build && cd build
cmake ..
make
```
This generates `libresonance_sdk.so`.

## How to Use (For Team HI)

Team HI **does not** need to write C++ or format JSON. They just use the `test_tx.py` file provided in the `test/` folder as a template.

### 1. The Valid Symbols Source
The `load_master_symbols()` function automatically dynamically parses the `master_symbols.json` file generated by the simulator. This ensures Team HI can *only* request mathematically verified patterns (e.g., `"CHLADNI_191"`).

### 2. Python Integration Example
Here is how Team HI interacts with the SDK inside their UI code:

```python
import ctypes

# 1. Load the compiled SDK library
tx_lib = ctypes.CDLL("./libresonance_sdk.so")

# 2. Setup the argument types for safety
tx_lib.init_zmq.argtypes = [ctypes.c_char_p]
# ... (see test_tx.py for all argtypes)

# 3. Connect to the Hardware Receiver via ZeroMQ
tx_lib.init_zmq(b"tcp://127.0.0.1:5555")

# 4. Create a buffer to receive the generated JSON
buffer = ctypes.create_string_buffer(512)

# 5. Send an action! (Triggered by a UI button)
# We hand the raw variables safely to the C++ SDK:
tx_lib.format_json(
    b"CHLADNI_191",           # Verified Symbol ID
    b"AUTO",                  # LED behavior
    b"FAST",                  # Transition speed
    ctypes.c_float(75.5),     # Volume
    ctypes.c_int(5),          # Musical Note
    b"sequence_1",            # Rhythm
    ctypes.c_int(120),        # BPM
    buffer,                   # C++ will populate this with strict JSON
    512
)

# 6. Send the generated buffer over the network
tx_lib.send_zmq(buffer.value)
```

## File Breakdown
*   **`include/mqdriver_tx.hpp`**: The C++ declarations. We use `extern "C"` to disable name mangling so Python can read the functions.
*   **`src/mqdriver_tx.cpp`**: The C++ implementation. Handles `snprintf` JSON building and ZeroMQ networking safely.
*   **`test/test_tx.py`**: The dummy Python implementation and reference implementation for Team HI.
